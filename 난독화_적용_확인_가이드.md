# 난독화 적용 확인 가이드 (직접 따라하기용)

이 순서대로 하나씩 진행하시면 됩니다. 중간에 이상한 게 나오면 그 화면을
그대로 캡처해서 저한테 보여주시면 됩니다 — 다음 단계로 넘어가지 마시고요.

---

## 0단계. 터미널 열고 프로젝트 폴더로 이동

터미널을 열고 아래를 그대로 복사해서 붙여넣고 엔터.

```
cd "/Users/han87/Desktop/네이버 블로그 자동화"
```

- 성공하면: 아무 메시지 없이 다음 줄로 넘어갑니다(정상).
- "No such file or directory" 같은 에러가 뜨면: 폴더 이름이나 위치가 바뀐
  것일 수 있어요 — 저한테 알려주세요.

---

## 1단계. 새로 필요해진 패키지 설치 (npm install)

```
npm install
```

- 이 명령은 이번에 추가한 `javascript-obfuscator`라는 도구를 실제로
  컴퓨터에 내려받아 설치하는 과정입니다. 인터넷 연결이 필요하고,
  1~3분 정도 걸릴 수 있습니다.
- **성공 화면**: 여러 줄의 진행 로그가 뜨다가 마지막에 보통
  `added N packages in Xs` 같은 문장으로 끝납니다. 중간에 노란색/주황색
  경고(`npm warn ...`)가 몇 줄 나오는 건 정상입니다 — 무시하셔도 됩니다.
- **실패 화면**: 빨간 글씨로 `npm error`가 뜨고 명령이 중간에 멈추면
  실패입니다 — 그 화면 전체를 캡처해서 저한테 보여주세요.

---

## 2단계. 난독화만 먼저 테스트 (전체 빌드 전에 빠르게 확인)

```
npm run obfuscate
```

- 이 명령은 전체 앱을 패키징하지 않고, 난독화 스크립트만 빠르게
  실행해서 잘 작동하는지 확인하는 용도입니다. 몇 초면 끝납니다.
- **성공 화면**: 아래와 비슷한 내용이 출력됩니다.
  ```
  [obfuscate] 시작 — 대상 파일: 5개
  [obfuscate] main.js: 366.0KB -> 900.0KB   (숫자는 다를 수 있음)
  [obfuscate] preload.js: 11.4KB -> 30.0KB
  [obfuscate] src/db.js: ...
  [obfuscate] license/licenseCore.js: ...
  [obfuscate] license/timeGuard.js: ...
  [obfuscate] 완료 -> build-obfuscated
  [obfuscate] 소스맵(비공개, 배포 미포함) -> build-obfuscated-maps
  ```
- **확인해보기(선택)**: Finder에서 프로젝트 폴더를 열어보면 `build-obfuscated`
  라는 새 폴더가 생겨있을 겁니다. 그 안의 `main.js`를 텍스트 편집기로
  열어보면(더블클릭 말고 우클릭 → 다른 앱으로 열기 → 텍스트 편집기)
  알아볼 수 없는 뒤섞인 글자들이 보일 거예요 — 그게 정상이고, 난독화가
  잘 됐다는 뜻입니다. **이 파일은 절대 직접 수정하지 마세요** — 다음 빌드
  때마다 자동으로 새로 만들어지는 결과물일 뿐입니다.
- **실패 화면**: `[obfuscate] 실패: ...` 라는 빨간 글씨가 뜨면 캡처해서
  저한테 보여주세요.

## 3단계. 실제 앱으로 패키징 (로컬 테스트용, 아직 배포 아님)

```
npm run build:mac
```

- **주의**: 이건 "업데이트 배포.command"가 아닙니다! 이 명령은 git에
  커밋/푸시하거나 실제 사용자에게 배포하지 않고, 내 컴퓨터 안에서만
  테스트용 앱을 만드는 것입니다. 안심하고 실행하셔도 됩니다.
- 시간이 꽤 걸립니다(첫 실행이면 5~15분 정도, 컴퓨터 사양에 따라 다름).
  중간에 화면이 멈춘 것처럼 보여도 정상이니 기다려주세요.
- **성공 화면**: 마지막 줄 근처에 `building        target=... file=dist/...`
  같은 문장이 몇 번 나오고 에러 없이 끝납니다.
- **실패 화면**: 빨간 글씨의 에러 메시지가 뜨고 중간에 멈추면 실패입니다
  — 화면 전체를 캡처해서 저한테 보여주세요(어디서 막혔는지가 중요합니다).

---

## 4단계. 만들어진 테스트 앱 직접 실행해보기

- Finder에서 프로젝트 폴더 → `dist` 폴더를 엽니다.
- 그 안에 `.app` 파일(예: "네이버 블로그 자동화.app")이나 `.dmg` 파일이
  있을 겁니다. `.dmg`가 있으면 더블클릭해서 열고, 그 안의 앱 아이콘을
  Applications 폴더로 드래그하거나 그냥 더블클릭해서 실행합니다.
- **평소 쓰시던 것처럼 아래 기능들을 한 번씩 확인해주세요:**
  - 앱이 에러 없이 켜지는지
  - 네이버 계정 로그인이 되는지
  - 글 생성이 되는지(주제/키워드 입력 → 생성)
  - 썸네일 자동 생성이 되는지
  - 발행이 정상적으로 되는지(테스트 계정으로)
  - 환경설정 화면들이 다 열리는지
  - 오류 로그 화면, 문의하기 버튼이 평소처럼 열리는지
- 뭔가 하나라도 평소와 다르게 동작하거나 에러가 뜨면, **그 증상을
  그대로 저한테 알려주세요** — 커밋/배포하지 마시고 여기서 멈춰주세요.

---

## 5단계. 문제없이 확인되면 — 코드 저장(git) + 정식 배포

전부 정상이면, 아래를 순서대로 실행해주세요.

```
cd "/Users/han87/Desktop/네이버 블로그 자동화" && git add -A && git commit -m "난독화 빌드 파이프라인 추가 (main.js/preload/license/src-db)" && git push
```

그 다음, 평소처럼 **"업데이트 배포.command"**를 더블클릭해서 정식
릴리즈를 진행하시면 됩니다. 이 스크립트가 실행하는 GitHub Actions
빌드 과정에서도 이번에 추가한 난독화가 자동으로 똑같이 적용됩니다 —
방금 4단계에서 직접 확인하신 것과 동일한 결과물이 만들어집니다.
