# 네이버 블로그 자동화 — 업데이트 이력

## v1.1.8 (2026-07-18) — 현재 버전

- **로그인 시 실제 네이버 아이디가 저장 안 되고 임시 아이디(`naver_숫자`)로 등록되던 문제 근본 수정** — 네이버 페이지 구조가 바뀌면서 기존에 아이디를 찾던 방식들이 더 이상 작동하지 않는 것을 확인, "내 블로그" 버튼을 실제로 눌러 이동한 주소에서 아이디를 가져오는 방식으로 재구현. 아이디를 정상적으로 찾아놓고도 창을 닫는 과정에서 그 결과가 다시 지워지던 내부 버그도 함께 수정.
- 아이디 형식을 확인할 때 하이픈(-)이 포함된 아이디를 잘못된 형식으로 오판정하던 문제 수정
- "전체 초기화" 버튼이 발행 이력·예약까지 통째로 지워, 반복 사용 시 하루 최대 발행 횟수 제한을 우회할 수 있던 문제 수정 — 이제 로그 기록·오류 로그·자동화 루프 로그 3가지만 초기화하도록 범위를 좁히고, 버튼 이름도 실제 동작에 맞게 **"로그 전체 초기화"**로 변경
- 계정 관리 화면 전면 개편
  - 계정이 많거나 항목이 길면 가로 스크롤이 항상 생기던 문제 해결 — 각 열 너비를 마우스로 직접 늘리거나 줄일 수 있고, 경계선을 더블클릭하면 내용에 딱 맞는 최소 너비로 자동 조정됨(엑셀과 동일한 방식)
  - 계정마다 있던 개별 삭제 버튼을 없애고, 상단의 작은 삭제 버튼으로 여러 계정을 체크해서 한 번에 삭제하는 방식으로 변경(오조작 방지 + 화면 공간 확보)
  - 계정 순서를 마우스로 끌어서 원하는 대로 바꿀 수 있는 기능 추가(재시작해도 유지됨)
  - 항목을 수정할 때 저장/취소 버튼을 화면 바깥으로 옮겨, 수정 중에도 가로 스크롤이 생기지 않도록 개선
- 썸네일 디자인 설정에 **"랜덤"** 옵션 추가 — 매 발행마다 22종 디자인 중 하나가 무작위로 적용됨("기본" 옵션 바로 옆에 위치)
- 자동화 루프 화면의 "계정당 카테고리 추가 배정(+추가)" 기능이 계정 2개(테스트용 시범 계정)에만 한정되어 있던 것을 확인, 등록된 계정 수와 무관하게 **모든 계정에서 사용 가능**하도록 확대
- "실제 카테고리 불러오기" 버튼이 일부 계정에서 실패하는 문제 조사 — 계정을 방금 새로 로그인한 직후에는 정상 작동하고, 로그인 후 시간이 어느 정도 지난 계정에서는 실패하는 것으로 확인됨(네이버 쪽에서 이 카테고리 관리 화면에 더 엄격한 최근 로그인 기준을 적용하는 것으로 추정). 관련 세션 처리 방식도 발행 기능과 동일하게(매번 새로운 임시 세션) 정비하고, 실패 시 원인을 더 정확히 남기도록 진단 로그 보강. 이 버튼이 안 될 경우 해당 계정으로 재로그인 후 다시 시도하면 됨

## v1.1.7 (2026-07-17)

- **완전자동 발행 시 실제로는 실패했는데 "발행 완료"로 잘못 표시되던 문제 수정** — 원인은 이전 세션에서 작성 중이던 글이 있을 때 네이버 에디터가 띄우는 "작성 중인 글이 있습니다" 안내 팝업이 실제 편집 화면을 가리고 있던 것으로 확인됨(수동 발행 때는 사람이 직접 팝업을 확인하고 취소를 눌러왔으나, 완전자동 모드는 이 팝업의 존재 자체를 인지하지 못해 그 뒤로도 붙여넣기·클릭을 계속 "시도"만 하고 실제로는 아무 것도 반영되지 않았음). 이제 팝업을 자동으로 감지해 취소 처리한 뒤 정상적으로 새 글 작성을 진행함.
- **발행 카테고리 이름이 네이버 자체 내장 기능("내돈내산 상품 첨부")과 우연히 같은 경우, 엉뚱하게 그 기능 버튼을 클릭해 관련 없는 동의창이 뜨고 이후 발행 패널 전체가 막히던 문제 수정** — 카테고리를 찾을 때 이 내장 기능 문구가 포함된 후보는 제외하도록 개선.
- 완전자동 루프 실행 중 특정 조건에서 발행이 되지 않고 오류로 중단되던 문제 수정 (내부 설정값 참조 오류)
- 네이버 에디터 하단에 "전체 글감" 패널이 추가된 뒤로, 발행 과정에서 태그가 엉뚱한 곳(글감 검색창 등)에 잘못 입력되던 문제 수정 — 발행 전 태그 입력 로직을 정비하고, 발행 패널이 항상 정상적으로 펼쳐지도록 이 패널을 자동으로 접어두는 처리 추가
- 이미지 자동 첨부 시, 같거나 비슷한 키워드로 여러 번 발행하면 매번 같은 사진이 반복 삽입되던 문제 완화 — 관련성 높은 상위 후보 안에서 매번 다른 조합이 무작위로 선택되도록 개선(엉뚱한 사진이 섞일 위험은 늘리지 않는 범위로 조정)
- 썸네일이 문서 맨 위쪽에 가까울 때 정렬 버튼이 상단 고정 툴바와 겹쳐 가운데 정렬이 간헐적으로 반영되지 않는 현상 확인 — 위험 구간 감지 시 보정 스크롤 후 재시도하도록 방어 로직 추가(정확한 재현 조건은 계속 조사 중)
- 썸네일이 정상적으로 만들어졌는데도 뒤늦게 "썸네일 생성 타임아웃" 경고가 남아 혼동을 주던 로그 오류 수정
- 로그 기록 화면에서 최근 200줄까지만 보이던 것을 1000줄로 확대 — 계정을 여러 개 완전자동으로 돌릴 때 이전 계정 기록이 밀려서 안 보이던 문제 해결
- "브라우저 표시" 옵션이 지금까지 수동 발행에만 적용되던 것을, 반자동(예약 발행 포함)·완전자동 모드에서도 사용할 수 있도록 확장 — 자동화 도중 실제 작업 화면을 직접 확인하고 싶을 때 켜둘 수 있음
- 발행 각 단계(본문 입력, 공개설정 등)의 재시도 여유를 늘려 완전자동 루프에서의 안정성 강화
- (개발자 전용) 개발자 도구가 개발자 버전 실행 시 자동으로 열리던 것을 기본 꺼짐으로 변경하고, 필요할 때 Ctrl+Alt+I로 직접 열고 닫을 수 있도록 변경 — 실제 배포판에는 포함되지 않음
- Windows 배포 빌드에 소스 난독화 파이프라인 적용 (macOS는 실행 환경(Electron) 이슈로 이번 버전에서는 배포 보류)
