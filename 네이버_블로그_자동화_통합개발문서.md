| 기술 | 역할 | 선택 이유 |

## 2단계 · 로그인 & 계정 관리 시스템 (2~3일)

### 로그인 방식 — 쿠키/세션 저장 방식

네이버의 자동화 감지를 피하기 위해 **최초 1회 수동 로그인 후 쿠키를 저장**하는 방식을 사용합니다.

```
최초 1회: 사용자가 앱 내 브라우저에서 직접 네이버 로그인
     ↓
앱이 로그인 쿠키를 로컬에 암호화하여 저장
     ↓
이후: 저장된 쿠키로 로그인 상태 자동 복원
     ↓
글 작성 & 발행 자동화 가능 (쿠키 만료 전까지 유지)
```

> **장점:** 네이버 입장에서 일반 사용자 브라우저와 구분 불가, 계정 정지 위험 대폭 감소

### 작업 내용
- [ ] 네이버 로그인 버튼 클릭 시 드롭다운으로 로그인할 계정 수 선택 (1~10개)
- [ ] 선택한 수만큼 Electron 내장 브라우저(Chromium) 로그인 창 순차 표시
- [ ] 사용자가 직접 로그인 완료 시 쿠키 자동 추출
- [ ] 계정별 쿠키 암호화 후 SQLite에 저장
- [ ] 계정 관리 화면 구현

### 계정 관리 화면 기능
- 등록된 계정 목록 표시 (아이디 / 닉네임 / 마지막 로그인 날짜 / 상태)
- 계정별 닉네임·메모 편집
- 계정 추가 / 삭제
- 로그인 상태 확인 (쿠키 유효 여부 자동 체크)
- 쿠키 만료 시 재로그인 알림

### 결과물
> ✅ 최초 1회 수동 로그인 후 자동 세션 유지 완성

### AI / 인간화 설정 (드롭다운)

환경설정 화면에서 아래 항목을 드롭다운으로 선택할 수 있습니다.

#### 🔹 문장 길이 스타일
```
드롭다운 선택:
├── 자동 (랜덤 변화)       ← 기본값, AI가 짧은 문장과 긴 문장을 자연스럽게 혼합
├── 짧은 문장 위주         ← 템포감 있고 읽기 쉬운 스타일
└── 긴 문장 위주           ← 상세하고 전문적인 스타일
```

#### 🔹 문체 스타일
```
드롭다운 선택:
├── 자동 (구어체+문어체 혼합)  ← 기본값, 가장 자연스럽고 인간적인 글
├── 구어체 위주                ← "~했어요", "~인데요" 일상 말투
└── 문어체 위주                ← "~합니다", "~됩니다" 격식체
```

#### 🔹 개인 경험담 삽입
```
드롭다운 선택:
├── 자동 삽입 (권장)   ← "저는 실제로 가봤는데...", "써보니까..." 등 자동 삽입
├── 많이 삽입          ← 경험담 비중 높임 (리뷰형 글에 적합)
├── 적게 삽입          ← 경험담 최소화 (정보형 글에 적합)
└── 삽입 안 함         ← 경험담 없이 순수 정보 중심
```

#### 🔹 글 톤 (기본값 설정)
```
드롭다운 선택:
├── 정보형    ← 객관적 정보 중심, "~입니다", 수치·데이터 포함
├── 일상형    ← 친근한 말투, 일기·후기 형식
├── 리뷰형    ← 장단점 분석, 별점·총평 포함
└── 감성형    ← 감정 표현 풍부, 분위기·느낌 위주
```

> 💡 환경설정의 값은 기본값으로 저장되며, 글 생성 화면에서 글마다 개별 변경도 가능합니다.

### 발행 안전 설정

- [ ] 중복 글 유사도 경고 기준 설정 (기본값: 70%)

- 키워드를 제목·소제목·본문에 자연스럽게 배치

- 개인 경험담 자동 삽입 (설정값 반영)
- 글 톤에 맞는 감정 표현 & 주관적 의견 추가

## 6단계 · 발행 & 스케줄러 (2~3일)

### 발행 방식 — 쿠키 세션 활용 자동 발행

```
저장된 계정 쿠키 불러오기
     ↓
Electron 내장 브라우저로 네이버 블로그 글쓰기 페이지 접속
     ↓
생성된 글 + 이미지 자동 입력
     ↓
해시태그 자동 입력
     ↓
발행 버튼 클릭 (또는 예약 발행 설정)
```

### 작업 내용
- [ ] 즉시 발행 기능
- [ ] 예약 발행 (날짜 + 시간 지정)
- [ ] 발행 스케줄러 화면 (캘린더 뷰)
- [ ] 1일 N회 자동 발행 설정
- [ ] 계정 로테이션 자동화 (1번→2번→3번 순환 발행)
- [ ] 발행 간격 랜덤화 (자연스러운 패턴 유지)
- [ ] 발행 완료 알림 (맥 알림센터 연동)

### 안전 장치
- 일일 발행 한도 초과 시 자동 중단 및 경고
- 중복 글 유사도 체크 후 경고
- 계정별 발행 간격 최소값 강제 적용

### 결과물
> ✅ 자동 발행 + 예약 스케줄러 완성

## 🔴 버그 수정 사항 (당시 우선순위 높음)

### 1. Windows 버전 — 해시태그 콤마(,) 누락
- **증상:** Windows 버전에서 글 생성 시 해시태그에 콤마(,) 구분자가 없음
- **결과:** 발행된 게시글에 태그가 등록되지 않음
- **원인 추정:** Windows / Mac 줄바꿈 처리 차이 또는 해시태그 파싱 로직 분기 문제
- **작업:** 해시태그 생성 로직에서 OS별 처리 확인 및 콤마 구분자 강제 적용

## 🟡 기능 추가 사항 (당시 계획)

### 2. 브라우저 켜고 끄기 토글 (헤드리스 모드)
- **위치:** 글 생성 버튼 좌측
- **기능:** 체크박스 또는 토글 스위치로 브라우저 표시 여부 선택
  - ✅ 체크 ON → 브라우저 화면 보이면서 발행 (디버깅/확인용)
  - ☐ 체크 OFF → 백그라운드 발행 (헤드리스 모드, 기본값)
- **구현:** Playwright headless 옵션을 해당 토글 값으로 동적 제어

  - Unsplash API에서 키워드 기반 대표 이미지 1장 선택

### 4. 글감 자동 수집 (리서치 자동화)
- **기능:** 카테고리에 키워드 1회 등록 → 정해진 주기마다 관련 글 자동 수집
- **수집 주기 선택:** 매일 / 6시간마다 / 12시간마다
- **활용:** 새 글 작성 시 수집된 관련 글을 참고 컨텍스트로 Groq에 전달 → 더 풍부한 글 생성
- **구현 방향:**

  - 글 생성 프롬프트에 수집 자료 요약본 자동 삽입

### 5. 네이버 에디터 적극 활용 — 인간형 게시글 서식 ⭐ 핵심

**당시 문제:** 순수 텍스트 붙여넣기 → 서식 없는 밋밋한 글
**목표:** 사람이 직접 작성한 것처럼 에디터 서식 완전 적용

#### 구현 방식: Groq 프롬프트에서 네이버 호환 HTML 직접 생성

글 생성 단계에서부터 네이버 에디터 호환 HTML 형식으로 출력하고,
네이버 에디터 소스코드(`{}`) 버튼을 통해 HTML을 주입하는 방식

#### 적용할 서식 요소

| 요소 | 설명 | 예시 |
|------|------|------|
| 인트로 박스 | 제목을 박스로 강조 | 둥근 테두리 박스 안에 제목 + 이모지 |
| 대주제 소제목 | 섹션 구분 | 왼쪽 컬러 선 강조 박스 |
| 중주제 소제목 | 세부 항목 | 배경색 박스 또는 볼드 텍스트 |
| 이모지 | 자연스러운 감성 표현 | 문단 시작/끝에 적절한 이모지 삽입 |
| 인용구 박스 | 핵심 문장 강조 | 따옴표 스타일 인용 박스 |
| 구분선 | 섹션 분리 | `<hr>` 또는 점선 구분 |
| 쿠팡 파트너스 고지 | 상단 필수 표시 | 손글씨 스타일 이미지 또는 텍스트 박스 |

#### 네이버 호환 HTML 구조 예시

```html

    이에 따른 일정액의 수수료를 제공받고 있습니다.
  </span>

바로 휴대용 블루투스 스피커예요.</p>

방수 등급도 IPX7이라 물에 담가도 끄떡없더라고요.</p>

<!-- 인용구 박스 -->
<div style="border:1px solid #ddd; border-radius:8px;
padding:16px 20px; background:#fafafa; margin:16px 0;
position:relative;">
  <span style="font-size:24px; color:#ccc; line-height:1;">"</span>
  <p style="margin:4px 0; color:#555;">핵심 특징 요약 문장을 여기에</p>
  <span style="font-size:24px; color:#ccc; line-height:1;">"</span>
</div>

<!-- 구분선 -->
<hr style="border:none; border-top:1px dashed #ddd; margin:24px 0;">
```
